# Структура

---

- services - конфиги ОС для авто запуска и слежки за состоянием основных процессов
        
        1. запуск и проверка wi-fi, в случае сбоя мигает лампочка на rp
        2. запуск основного цикла опроса
        3. запуск клиента

- config-templates
    
        1. system.yaml - основные настройки системы с адресом бэка, путями логов итп
        2. user_config.yaml - все то, что может ввести пользователь

- main_question_loop
        
        основной щикл опроса, следит за датчиками, кладет метрики в файл из другого файла читает команды бэка

- client 

        клиент, общается с бэком

- setups
    
        скрипты сетапов

клиент и оцо в отдельных процессах, изолированны друг от друга, общаются через файлы, во избежания дедлоков, ебли с сокетами
итп, в случае падения одного его перезапускает ОС через сервисы, при этом если падает оцо с ошибкой, пишет это в метрики перед смертью
 клиент шлет алерт на бэк, если мертв клиент, бэк не пингует его и шлет алерт юзеру, если нету wifi - same
