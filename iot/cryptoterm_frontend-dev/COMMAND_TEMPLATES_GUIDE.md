# Создание шаблонов команд ASIC

## Обзор

Новый функционал для администраторов позволяет создавать, просматривать и управлять шаблонами команд для ASIC майнеров через веб-интерфейс.

## Компоненты

### 1. **CommandTemplateCreator** 
Компонент для создания новых шаблонов команд с предзаполненными полями.

**Функции:**
- Предзаполненный JSON шаблон для смены режима мощности
- 13 предустановленных режимов (от 3495W до 7700W)
- Быстрый выбор режима одним кликом
- Редактирование всех параметров (модель, производитель, прошивка, пароль)
- Предпросмотр JSON перед созданием
- Автоматическое формирование имени шаблона: `{watts}W_{hashrate}TH`

**Поля:**
- Мощность (Watts) - обязательное
- Хешрейт (TH/s) - обязательное
- Имя шаблона - опциональное (генерируется автоматически)
- Модель майнера - обязательное (по умолчанию: Antminer S19 Pro Hydro)
- Производитель - обязательное (по умолчанию: Bitmain)
- Прошивка - выбор из списка (anthill, vnish, stock)
- Пароль ASIC - для авторизации на устройстве

### 2. **CommandTemplateManager**
Компонент для управления существующими шаблонами.

**Функции:**
- Просмотр списка всех шаблонов
- Создание новых шаблонов (интеграция с CommandTemplateCreator)
- Просмотр деталей шаблона (JSON)
- Удаление шаблонов
- Обновление списка

## API методы

Добавлены следующие методы в `api.js`:

```javascript
// Создать шаблон (только администратор)
api.createAsicCommandTemplate(template)

// Получить все шаблоны
api.getAllCommandTemplates()

// Получить шаблон по имени
api.getCommandTemplate(templateName)

// Обновить шаблон (только администратор)
api.updateCommandTemplate(templateName, template)

// Удалить шаблон (только администратор)
api.deleteCommandTemplate(templateName)
```

## Использование

### Доступ к интерфейсу

1. Войдите в систему как администратор
2. На главной странице Dashboard перейдите на вкладку **"🔧 Команды"**
3. Нажмите **"➕ Создать новый шаблон"**

### Создание шаблона (быстрый способ)

1. Выберите один из предустановленных режимов (например, `3495W_132TH`)
2. При необходимости отредактируйте параметры
3. Нажмите **"✓ Создать шаблон"**

### Создание кастомного шаблона

1. Введите мощность (Watts)
2. Введите хешрейт (TH/s)
3. Укажите модель майнера
4. Укажите производителя
5. Выберите прошивку
6. Нажмите **"👁 Показать JSON"** для предпросмотра
7. Нажмите **"✓ Создать шаблон"**

### Просмотр и удаление шаблонов

1. На вкладке **"🔧 Команды"** увидите список всех шаблонов
2. Нажмите **"👁 Детали"** для просмотра полного JSON
3. Нажмите **"🗑 Удалить"** для удаления шаблона

## Логика работы шаблона

Каждый созданный шаблон выполняет 3 шага:

1. **unlock** - Авторизация на ASIC:
   - POST запрос на `/api/v1/unlock` с паролем
   - Получение токена и сохранение в переменную `${token}`

2. **set_profile_{watts}** - Установка профиля мощности:
   - POST запрос на `/api/v1/settings` с токеном авторизации
   - Установка preset режима мощности

3. **restart_mining** - Перезапуск майнинга:
   - POST запрос на `/api/v1/mining/restart` для применения изменений

## Backend API

Шаблоны создаются через эндпоинт:
```
POST /api/admin/asic-command-templates
```

После создания шаблоны доступны для выполнения на устройствах через:
```
POST /api/v1/asic-commands/execute?rpId={rpId}&asicId={asicId}&commandId={templateName}
```

## Безопасность

- Создание/редактирование/удаление шаблонов доступно только администраторам
- Требуется роль `ADMIN` в JWT токене
- Просмотр доступных команд доступен всем авторизованным пользователям

## Файлы

- `/src/components/CommandTemplateManager.jsx` - Управление шаблонами
- `/src/components/Dashboard.jsx` - Интеграция в главное меню
- `/src/api.js` - API методы для работы с шаблонами

## Примечания

- Пароль ASIC сохраняется в шаблоне и используется при каждом выполнении команды
- Шаблоны привязаны к конкретной модели майнера через поле `minerModel`
- При выполнении команды система проверяет совместимость модели устройства с моделью в шаблоне
- IP адрес и порт ASIC определяются автоматически на Raspberry Pi по ID устройства
