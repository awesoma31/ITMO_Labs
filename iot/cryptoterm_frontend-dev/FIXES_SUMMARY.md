# Исправления - Сводка изменений

## Исправленные проблемы

### 1. ✅ Username в Dashboard
**Проблема:** Отображался ID пользователя вместо username  
**Решение:**
- Добавлены методы `getUsernameFromToken()` и `getEmailFromToken()` в `auth.jsx`
- Dashboard теперь извлекает username из JWT токена
- Если username = "anonymous", показывается часть email или userId

**Файлы:**
- `src/auth.jsx` - добавлены новые методы
- `src/components/Dashboard.jsx` - обновлена логика получения username

### 2. ✅ Курс валют BTC/USD
**Проблема:** Показывался некорректный курс (89.02 вместо реального)  
**Решение:**
- Бэкенд теперь возвращает реальный курс BTC/USD в `MiningProfitResult`
- Добавлены поля `btcPriceUsd` и `totalPowerConsumption` в DTO
- Фронтенд использует курс из API вместо hardcoded значения

**Файлы Backend:**
- `MiningProfitResult.java` - добавлены поля `btcPriceUsd` и `totalPowerConsumption`
- `MiningProfitService.java` - возвращает реальный курс из BtcNetworkService

**Файлы Frontend:**
- `src/components/ProfitCalculator.jsx` - использует `btcPriceUsd` из API

### 3. ✅ График хэшрейта с несколькими линиями
**Проблема:** График показывал только суммарный хэшрейт  
**Решение:**
- График теперь показывает отдельную линию для каждого устройства
- Добавлена легенда с цветовой индикацией
- При клике на легенду можно скрывать/показывать линии устройств
- Каждое устройство имеет свой цвет

**Файлы:**
- `src/components/HashRateChart.jsx`:
  - Загружает данные для каждого устройства отдельно
  - Объединяет данные по timestamp
  - Рендерит несколько `<Line>` компонентов
  - Добавлен `CustomLegend` с кликабельными элементами
  - Состояние `visibleDevices` управляет видимостью линий

### 4. ✅ Размер графика при загрузке

### 6. ✅ Панель логов (Developer Panel)
**Проблема:** Отсутствовала панель для просмотра логов  
**Решение:**
- Добавлена секция "Device Logs (Developer Panel)" внизу Dashboard
- Табы для выбора устройства
- Интеграция компонента `ConsoleLogs`
- Временные диапазоны: 5 min, 15 min, 1 hour, 6 hours, 1 day
- Кастомный выбор времени (from/to)
- Цветовая индикация уровней логов:
  - ERROR/FATAL - красный
  - WARN - желтый
  - INFO - зеленый
  - DEBUG - фиолетовый

**Файлы:**
- `src/components/Dashboard.jsx` - добавлена секция логов
- `src/components/ConsoleLogs.jsx` - компонент для отображения логов
- `src/api.js` - добавлен alias `getMinerLogs()`

### 7. ✅ Логи парсятся корректно
**Проблема:** Логи доходили до фронта но неверно парсились  
**Решение:**
- `ConsoleLogs` теперь корректно обрабатывает структуру `DeviceLogDto`
- Поля: `time`, `level`, `message`
- Форматирование времени в читаемый вид
- Нумерация строк логов

**Backend DTO:**
```java
public class DeviceLogDto {
    private OffsetDateTime time;
    private String level;
    private String message;
}
```

## Тестирование

Для тестирования исправлений:

1. **Username:**
   - Войдите в систему
   - Проверьте, что показывается корректный username (не UUID)

2. **Курс BTC:**
   - Проверьте калькулятор прибыли
   - Курс должен быть около 78,000-100,000 USD

3. **График:**
   - Убедитесь, что показывается несколько цветных линий
   - Кликните на легенду - линия должна скрываться/показываться
   - Hover на график - tooltip должен показывать все устройства

4. **Логи:**
   - Прокрутите вниз до секции "Device Logs"
   - Выберите устройство
   - Выберите временной диапазон
   - Проверьте, что логи отображаются с цветовой индикацией

## Известные ограничения

1. Расчет потребления энергии использует примерное значение 3.5 kW на майнер
2. График показывает до 7 устройств с уникальными цветами (дальше цвета повторяются)
3. Логи загружаются синхронно (могут быть медленными для больших диапазонов)
