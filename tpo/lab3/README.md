## Описание тестовых сценариев

### TC-01 — Поиск возвращает результаты
- **Предусловие:** market.yandex.ru доступен
- **Шаги:** открыть главную страницу → ввести «ноутбук» → нажать «Найти»
- **Ожидаемый результат:** страница результатов содержит карточки товаров (`article[data-auto=productCard]`)

### TC-02 — Запрос отражён в URL
- **Предусловие:** market.yandex.ru доступен
- **Шаги:** открыть главную страницу → ввести «смартфон» → нажать «Найти»
- **Ожидаемый результат:** URL содержит строку `смартфон` или её URL-encoded эквивалент

### TC-03 — Поиск без результатов
- **Предусловие:** market.yandex.ru доступен
- **Шаги:** открыть главную страницу → ввести случайную строку «zzxqwerty12345нетакоготовара9999»
- **Ожидаемый результат:** карточки товаров отсутствуют ИЛИ отображается сообщение «Ничего не нашлось»

### TC-04 — Названия товаров присутствуют
- **Предусловие:** поиск «телевизор» возвращает результаты
- **Шаги:** открыть главную → ввести «телевизор» → дождаться результатов
- **Ожидаемый результат:** найден хотя бы один непустой заголовок товара (`h3[data-auto=snippet-title]`)

### TC-05 — Цены отображаются
- **Предусловие:** поиск «наушники» возвращает результаты
- **Шаги:** открыть главную → ввести «наушники» → дождаться результатов
- **Ожидаемый результат:** найдена хотя бы одна цена (`span[data-auto=mainPrice]`)

### TC-06 — Кнопка сортировки по цене присутствует
- **Предусловие:** поиск «ноутбук» возвращает результаты
- **Шаги:** открыть главную → ввести «ноутбук» → дождаться результатов
- **Ожидаемый результат:** на странице видна кнопка/ссылка «по цене»

### TC-07 — Страница товара содержит заголовок
- **Предусловие:** поиск «планшет» возвращает хотя бы один товар
- **Шаги:** поиск «планшет» → нажать на первый результат
- **Ожидаемый результат:** страница товара загружена, `h1` с непустым текстом виден

### TC-08 — Страница товара отображает цену
- **Предусловие:** поиск «монитор» возвращает хотя бы один товар
- **Шаги:** поиск «монитор» → нажать на первый результат
- **Ожидаемый результат:** на странице товара видна цена

## Технические решения

### PageObject Pattern
Каждая страница сайта представлена отдельным классом. Тесты работают только с методами страниц, не зная об XPath напрямую.

```
BasePage          ← абстрактный базовый класс (wait, isPresent, ...)
├── MainPage      ← open(), search()
├── SearchResultsPage ← waitForResults(), hasResults(), openFirstProduct(), ...
└── ProductPage   ← hasTitle(), getTitle(), hasPrice(), hasAddToCartButton(), ...
```

### XPath с data-auto атрибутами
Яндекс добавляет `data-auto` атрибуты специально для автоматизации — они стабильнее автогенерируемых CSS-классов. Каждый локатор содержит несколько fallback-вариантов через `|`:

```java
// Пример из SearchResultsPage.java
private static final By PRODUCT_CARDS = By.xpath(
    "//article[@data-auto='productCard']"
    + " | //div[@data-auto='snippet-cell']"
    + " | //li[contains(@data-auto,'productCard')]"
);
```

### Явные ожидания вместо sleep()
```java
// BasePage.java
protected WebElement waitVisible(By locator) {
    return wait.until(ExpectedConditions.visibilityOfElementLocated(locator));
}

// SearchResultsPage.java — ждём появления результатов ИЛИ сообщения "ничего не найдено"
wait.until(ExpectedConditions.or(
    ExpectedConditions.presenceOfElementLocated(PRODUCT_CARDS),
    ExpectedConditions.presenceOfElementLocated(NO_RESULTS)
));
```

### Параллельный запуск Chrome + Firefox
`@ParameterizedTest @MethodSource("browsers")` создаёт по экземпляру теста для каждого браузера. JUnit Platform запускает их конкурентно согласно `junit-platform.properties`:

```properties
junit.jupiter.execution.parallel.enabled=true
junit.jupiter.execution.parallel.mode.default=concurrent
junit.jupiter.execution.parallel.config.strategy=fixed
junit.jupiter.execution.parallel.config.fixed.parallelism=4
```

## Запуск тестов

```bash
# Запуск всех тестов (браузеры открываются видимыми — меньше риск капчи)
gradle test

# Headless режим (для CI или фонового запуска)
gradle test -Dheadless=true

# Просмотр отчёта
open build/reports/tests/test/index.html
```

### Требования
| Компонент | Версия |
|-----------|--------|
| JDK | 17+ |
| Gradle | 9.x |
| Chromium + chromedriver | любая совместимая |
| Firefox | любая (geckodriver скачается автоматически через Selenium Manager) |

> **Примечание:** Яндекс.Маркет использует защиту от ботов. При запуске в headless-режиме возможно появление CAPTCHA, из-за чего тесты завершатся ошибкой ожидания. В таком случае запустите без флага `-Dheadless=true`.
